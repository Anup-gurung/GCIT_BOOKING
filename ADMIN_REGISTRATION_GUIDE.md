# Admin Pre-Registration Guide

This guide explains how to pre-register admin users in the Arena Hub platform.

## Overview

You can pre-register admin accounts in three ways:

1. **Web UI** - Through the admin registration page (easiest)
2. **API** - Direct HTTP API calls
3. **Server Actions** - Using TypeScript functions

---

## Method 1: Web UI (Recommended)

The simplest way to pre-register admins is through the web interface.

### Steps:

1. **Login as an existing admin** to your Arena Hub application
2. **Navigate to**: `http://localhost:3000/admin-registration`
3. **Fill in the form**:
   - Email: Admin's email address
   - Full Name: Admin's display name
   - Password: (Optional - auto-generated if left empty)
4. **Add Multiple Admins**: Click "Add Admin" multiple times to add multiple accounts
5. **Register All**: Click "Register All Admins" to create all accounts at once

### Features:
- ✅ Bulk registration (add multiple admins at once)
- ✅ Auto-generate secure passwords
- ✅ Copy passwords to clipboard
- ✅ See registration results with status
- ✅ Error handling and notifications

---

## Method 2: API Endpoint

Send a POST request to register admins programmatically.

### Endpoint:
```
POST /api/admin/register
```

### Requirements:
- Must be authenticated (Bearer token)
- User must have admin role

### Request Body:
```json
{
  "admins": [
    {
      "email": "admin1@example.com",
      "name": "Admin One",
      "password": "YourPassword123!"
    },
    {
      "email": "admin2@example.com",
      "name": "Admin Two"
    }
  ]
}
```

### Example cURL:
```bash
curl -X POST http://localhost:3000/api/admin/register \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{
    "admins": [
      {
        "email": "neadmin@example.com",
        "name": "New Admin",
        "password": "SecurePass123!"
      }
    ]
  }'
```

### Response:
```json
{
  "results": [
    {
      "email": "newadmin@example.com",
      "name": "New Admin",
      "success": true,
      "password": "AutoGeneratedPassword123!"
    }
  ]
}
```

## Method 3: Server Actions

Use TypeScript functions directly from your app.

### Usage:

```typescript
import { createAdminUser, preRegisterAdminsByEmail } from '@/app/actions/admin-actions';

// Register single admin
const result = await createAdminUser(
  'admin@example.com',
  'SecurePassword123!',
  'Admin Name'
);

// Register multiple admins
const results = await preRegisterAdminsByEmail([
  { email: 'admin1@example.com', name: 'Admin One', password: 'Pass123!' },
  { email: 'admin2@example.com', name: 'Admin Two' }, // Password auto-generated
]);
```

## Bulk Registration with JSON File

Use the provided `admins-template.json` file:

### Steps:

1. **Edit** `admins-template.json` with your admin details:
```json
[
  {
    "email": "admin@example.com",
    "name": "Admin Name",
    "password": "SecurePassword123!"
  }
]
```

2. **Copy content** from `admins-template.json`
3. **Paste** into the admin registration page
4. **Click** "Register All Admins"

## Security Best Practices

### 1. Password Management
- ✅ Use strong passwords (min 12 characters)
- ✅ Include uppercase, lowercase, numbers, symbols
- ✅ Let the system auto-generate if unsure
- ❌ Don't share passwords in plain text
- ❌ Don't use simple passwords

### 2. Admin Access
- ✅ Only existing admins can create new admins
- ✅ Admins can view all bookings, referees, and users
- ✅ Each admin gets a unique auth token
- ❌ Don't share admin credentials

### 3. Email Confirmation
- All admin emails are automatically confirmed
- They can log in immediately after registration

## Database Schema

Admins are stored in two tables:

### auth.users (Supabase Auth)
```sql
id: UUID (primary key)
email: STRING
encrypted_password: STRING
```

### public.profiles
```sql
id: UUID (references auth.users)
name: STRING
role: TEXT (enum: 'user', 'admin')
created_at: TIMESTAMP
```

## Troubleshooting

### "User already exists"
- The email is already registered
- Solution: Use a different email or check if account exists

### "Only admins can create other admins"
- Current user is not an admin
- Solution: Log in with an admin account

### "Invalid token"
- Session expired or invalid authentication
- Solution: Log out and log back in

### Password not copied
- Check if clipboard access is enabled
- Try copying manually from the results screen
